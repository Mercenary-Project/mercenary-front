# AGENTS.md

이 파일은 `/workspace/Mercenary-Frontend` 하위 전체 경로에서 동작하는 CLI 에이전트를 위한 작업 규칙입니다.

## 1) 응답/출력 기본 틀
- 최종 답변은 아래 순서를 기본으로 사용한다.
    1. **Summary**: 변경사항 요약 (불릿)
    2. **Testing**: 실행한 검증 명령과 결과
    3. **Next Steps (Optional)**: 후속 제안 1~3개
- 테스트/검증 명령은 상태 이모지를 붙인다.
    - ✅ 성공
    - ⚠️ 환경 제약으로 경고
    - ❌ 실패
- 사실 확인이 필요한 설명에는 파일 경로와 라인 근거를 남긴다.

## 2) 건드리면 안 되는 것 (보호 규칙)
- 사용자 요청 없이 다음 항목은 수정/삭제/이동하지 않는다.
    - `.git/` 내부 파일
    - 배포/운영 시크릿 또는 키 파일 (`*.pem`, `*.key`, `.env` 실제값)
    - 잠금 파일 (`package-lock.json`, `pnpm-lock.yaml`, `yarn.lock`)의 대규모 재생성
- 의도하지 않은 대규모 포맷팅 변경(전체 파일 일괄 정렬/개행 수정)은 금지한다.
- 기존 동작을 바꾸는 리팩터링은 **명시적 요청이 있을 때만** 수행한다.

## 3) 코드/작업 룰
- 변경은 항상 최소 범위로 수행한다.
- 한 작업 단위마다 다음을 지킨다.
    - 수정 전: 관련 파일/규칙 확인
    - 수정 후: 가능한 범위에서 lint/test/build 중 1개 이상 검증
    - 결과 보고: 무엇을 왜 바꿨는지 한 줄 근거 포함
- 새 파일 생성 시 파일 상단에 목적이 드러나는 이름/구조를 사용한다.
- 디버깅용 코드(`console.log`, 임시 주석)는 커밋 전에 제거한다.

## 4) Git 규칙
- 커밋 메시지는 아래 형식을 권장한다.
    - `feat: ...`, `fix: ...`, `docs: ...`, `chore: ...`
- 하나의 커밋에는 하나의 논리적 변경만 담는다.
- 커밋 전 `git diff --staged`로 의도치 않은 변경이 없는지 확인한다.

## 5) 문서 작성 규칙
- README/문서에는 다음 정보를 우선 포함한다.
    - 실행 방법
    - 개발 규칙
    - 폴더 구조
    - 자주 발생하는 이슈와 해결법
- 예시 명령은 복사-실행 가능 형태로 제공한다.

## 6) 우선순위
1. 시스템/개발자/사용자 직접 지시
2. 더 하위 경로의 `AGENTS.md`
3. 이 문서

충돌 시 상위 우선순위를 따른다.
